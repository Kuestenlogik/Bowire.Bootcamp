# Unit 0: Foundations & setup

*Time: ~30 minutes • Lessons: 4 • Modality: onboarding (concepts + get running)*

Understand what Bowire is, learn the **two deployment shapes as a concept**, **get a workbench running**, and see how the bootcamp is organised into **courses** and **units**. Every course starts here — by the end you have Bowire open in your browser, so the units after this can assume it's up.

## Lessons

| Lesson | Topic | What You'll Learn |
|--------|-------|-------------------|
| [0.1](lesson-1/README.md) | What is Bowire? | Multi-protocol API workbench; how it positions vs Postman / Insomnia / Bruno; when to reach for it |
| [0.2](lesson-2/README.md) | The two deployment shapes | Standalone **CLI** vs **embedded** host — the mental model and the decision criteria |
| [0.3](lesson-3/README.md) | Get Bowire running | The least-ceremony path to an open workbench (embedded sample / CLI / embed in your host) |
| [0.4](lesson-4/README.md) | How this bootcamp works | Courses → units → lessons; one modality per unit; how to pick your course |

## Why this unit

Before any course you need two things: a clear mental model (*what Bowire is*, *the two shapes*) and a **running workbench** — you have to set up *somewhere*, and Unit 0 is that somewhere, before the courses branch. Setup here is deliberately minimal onboarding; the deep CLI story is [Unit 3](../unit-3/README.md) and the deep embedded story is [Unit 4](../unit-4/README.md).
